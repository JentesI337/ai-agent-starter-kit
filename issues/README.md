# Issues

Dieser Ordner dient als zentrale Ablage für projektbezogene Issues auf Root-Ebene.

## Zweck

- Sammeln und strukturieren von offenen Aufgaben, Bugs und Verbesserungen.
- Dokumentation von Status, Priorität, Kontext und Entscheidungen.
- Nachvollziehbare Historie für Planung und Umsetzung.

## Empfohlene Struktur

- Eine Datei pro Issue, z. B. `issue-0001-kurztitel.md`.
- Optional Unterordner pro Themenbereich (`backend`, `frontend`, `infra`) bei steigendem Umfang.

## Empfohlenes Template pro Issue

```markdown
# <Titel>

## Meta
- ID: issue-0001
- Status: open | in-progress | blocked | done
- Priorität: low | medium | high | critical
- Owner: <Name>
- Erstellt: YYYY-MM-DD
- Zuletzt aktualisiert: YYYY-MM-DD

## Kontext
<Kurze Problem- und Zielbeschreibung>

## Akzeptanzkriterien
- [ ] Kriterium 1
- [ ] Kriterium 2

## Umsetzung
<Geplante oder umgesetzte Schritte>

## Verifikation
<Tests, Checks, Nachweise>

## Notizen
<Offene Punkte, Entscheidungen, Links>
```

## Hinweis

Der Ordner ist bewusst leichtgewichtig gehalten und kann bei Bedarf an bestehende Projektprozesse angepasst werden.
